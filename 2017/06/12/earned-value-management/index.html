挣值管理原理与思考

发表于 2017-06-12 | 分类于 项目管理

挣值管理原理

挣值(Earned Value)管理是一种工作绩效分析方法。它通过比较计划工作量、挣值与实际花费成本来评价项目执行的成本和进度绩效。

挣值管理希望达到的目的是在项目管理过程中，通过量化手段来回答“项目已经完成了多少工作、进度和费用状况如何”的问题。

在运用挣值管理模型时，有五项必要的参数，用通俗的语言解释就是：

  • 我本来打算用多少钱全部做完这个事？即完工预算(Budget at Complete, BAC)
  • 我本来打算用多长时间全部做完这个事？即完工预计工期(Base Date At Completion, BDAC)
  • 截止目前，如果进度顺利，我应该花了多少钱？即计划价值(Planned Value, PV)
  • 截止目前，我实际已经花了多少钱？即实际费用(Actual Cost, AC)
  • 截止目前，我实际已经干的进度是我原本打算花多少钱干的？也就是所谓的挣值(Earned Value, EV)

举个例子：

比如我们计划投入600万元，用6个月时间铺设一段管道。在给出这个计划时，其实BAC和BDAC已经确定了，分别是

BAC = 600万

BDAC = 6个月

所以一般地BAC和BDAC是不需要计算就可以从计划中直接获取到的参数。假如按照计划我们应该完成铺设管道50%的工作量时，我们实际只完成了40%的工作量，花费了200万元，这个时候就可以用挣值分析了。

我们目前实际花的钱(Actual Cost)是200万元。

AC = 200万

但如果工作进度顺利，我们本来应该是完成50%的工作，应该花费的钱(Planned Value)就是600万*50%，即300万元。所以也能很直接地计算出

PV = 300万

最后就是计算我们目前实际已经完成的工作，是我们计划用多少钱来完成的，也就是计算挣值(Earned Value)。例子中我们完成了40%的工作量，按照计划应该是用600万*40% = 240万元来完成。

EV = 240万

此时，挣值管理模型的5个参数值已经全部有了。

从字面上理解，英文单词earned的意思是“挣得的，赚得的”，我们实际花费(actual cost)了200万的成本，赚(earn)到了240万的价值，这笔买卖是不是赚了？

但是挣值分析并不只是为估算成本服务的，同时我们要意识到现在的进度其实是落后的。现在我们用上面的五个值来回答两个问题：

  1. 铺设管道的工作什么时候能完工? (估计完工工期, EDAC)
  2. 完工总共要花多少钱? (估计完工成本, EAC)

从导致进度偏差的原因具备一般性的前提下来看，我们原来打算干50%，现在只干了40%，那么我们做事情的效率(进度绩效指数, SPI)其实是比计划要打折的:

SPI = EV/PV = 240/300 = 0.8

其实也可以从另一个方面来计算SPI：我们原计划干50%的，现在只干了40%，效率岂不就是打了8折吗？

这样来看的话，那么我们原计划6个月来完成的，既然现在的做事效率是原计划的80%，那么这个完工工期就要延长了：

EDAC = BDAC/SPI = 6个月/0.8 = 7.5个月

工期算出来了，那么如何算钱呢？

同样我们默认地假设成本的偏差也具备一般性。我们本来打算花240万干完40%的工作，现在我们只花了200万就干完了40%的工作，那么我们的资金效率(成本绩效指数, CPI)是要比计划中的要高的:

CPI = EV/AC = 240/200 = 1.2

我们回过头来把SPI和CPI放在一起看，其实它们本质上分别就是”时间的效率”与”资金的效率”，只不过这两个效率值都是用资金为参数来计算的，所以没有那么直观而已。

既然我们原计划中的1元钱能做1.2元的事情，那么原计划的600万其实也就用不完啦！

EAC = BAC/CPI = 600万/1.2 = 500万

此时我们就可以回答，项目完工500万就够了，但工期要延长到7.5个月。所以，我们在项目进行到中途的时候，可以计算到项目完工时的时间成本与资金成本，这就是挣值管理的作用。顺便，我们还得到了两个中间值，分别是进度绩效指数与成本绩效指数。也即做事的时间效率，与资金效率。

挣值管理模型除了能够预测完工成本与计算效率之外，还可以计算偏差，偏差的计算就更加简单了：

  1. 做到目前这个进度，我原本打算花多少钱，现在实际花了多少钱。这两者的差值叫成本偏差(CV)

    CV = PV - AC

  2. 做完这项工作，我原本打算花多少钱，现在来看我将会花多少钱。这两者的差值叫完工成本偏差(VAC)

    VAC = BAC - EAC

  3. 在目前这个时间点，我实际完成了多少工作，我原本打算完成多少工作。这两者的差值叫做进度偏差(SV). 这里需要注意的是“进度”这个概念在挣值管理中是以“资金”这个指标来衡量的，所以SV的计算方式是用EV减去PV

    SV = EV - PV

在挣值管理中，最不直观的概念其实就是“挣值”，它表示“现在完成的这些工作我原本是打算用多少钱来完成的”,这样表示的作用是为了让进度与资金之间建立桥梁。理解了挣值的概念，整个挣值管理模型就很容易理解了。

需要注意的是，我们在计算EDAC与EAC的时候，是假设了导致当前偏差的原因是具备一般性的。也就是说我们用实际的工作来重新计算了我们的时间效率与时间效率，来修正计划中就存在的偏差。如果导致当前偏差的原因不具备一般性，比如说在上面铺设管道的例子中，进度迟缓的原因是因为台风导致的，而刮台风这件事明显不具备一般性，就不能用修正CPI和SPI的方式来估算完工工期及完工成本了。

挣值管理在应用中的困难

挣值管理模型在实际应用中可能遇到一些困难，首先就是如何准确地给出当前完工的工作占总工作的百分比。类似铺设管道的工作，可以用管道长度来计算工作进度，但是对于含有研究与探索性质的工作，则很难统计已完成工作的占比。

另一个困难就是完成工作量与成本费用之间如何建立关系。比如IT项目建设，实现同样数量的接口，所需要投入的时间成本之间的差异可能是巨大的。即使是铺设管道这样的工作可以用长度来量化的工作，在市区施工与郊区施工的人工成本差异也可能是很大的。

最后我们再对挣值管理中的几个术语进行一个整理：

五个输入值

中文名称 英文缩写 英文名称 解释
完工预算 BAC Budget at Complete 项目总的成本预算
完工预计时间 BDAC Base Date At Completion 项目全部工作的预计时间
计划价值 PV Planned Value 现在应该做多少工作(用费用来衡量)
实际费用 AC Actual Cost 现在实际用了多少费用
挣值 EV Earned Value 现在实际做了多少工作(用费用来衡量)

七个输出值

中文名称 英文缩写 英文名称 解释
进度绩效指数 SPI Schedule Performance Index 做事的效率(用费用来衡量)
成本绩效指数 CPI Cost Performance Index 资金的效率
完工估算 EAC Estimate At Completion (按照现在的表现估算)完工需要的总费用
完工工期估算 EDAC Estimate Date At Completion (按照现在的表现估算)完工需要的总时间
成本偏差 CV Cost Variance 实际的工作与实际费用的差值
进度偏差 SV Schedule Variance 实际的工作与预期完成的工作的差值
完工费用偏差 VAC Variance At Completion 完工预算与完工估算之间的差值

学习建议

这部分公式比较多，直接记忆这些公式是非常低效的一件事，不如弄懂每一个术语的含义，然后公式自己就可以推导出来了。
